# Configurações (Config)

A pasta `config` contém classes de **configuração** da aplicação.

---

## Responsabilidades principais

- Configurar CORS, beans globais, serialização, etc.
- Centralizar configurações específicas de frameworks ou bibliotecas
- Deixar claro, em um único lugar, configurações que impactam todo o projeto

---

## Exemplos comuns

- Configuração de CORS para permitir/domínios específicos
- Configuração de conversores de data/hora
- Configuração de interceptors, filters globais, etc.

---

## Futuras modificações

- Adicionar novas configs conforme novos requisitos surgirem
- Manter as configurações organizadas para evitar confusão
